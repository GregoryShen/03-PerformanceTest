01. 性能测试的基本原理（步骤）是什么？
1) 模拟各种业务场景下的负载压力
2）观察压力下服务端业务逻辑处理是否正常
3）观察压力下服务端资源使用情况是否吃紧
4）改变模拟的压力来重复上述观察

02. 性能测试结果由什么组成
性能测试结果由请求结果与服务端监控结果组成

03. 请求结果包括什么？
包括响应时间、返回值等单次请求的响应里得到的结果以及他们的统计结果

04. 请求结果有什么特点？
请求结果是一个一个互相独立的结果，每个请求有自己的请求结果

05. 服务端监控结果包括什么？
包括服务端各项资源使用率及数据量等统计结果，具体包括一段时间内 CPU 的负载、 CPU 的使用
率、网络数据传输量、内存的占用率、IO数据量等指标变化的情况

06. 单次结果与统计结果有什么区别？
单次结果： 调用正确/调用错误
统计结果： 持续调用n小时或n次，其中调用正确比例为x%，叫做正确率或错误率(1-x%)

单次结果的维度有很多，不局限于正确错误率。
统计结果还包括通过对统计结果进行计算得到的新的统计结果

07. 总并发用户数指什么？
同一时间在系统上的用户数量，这些用户可能分布在不同的功能模块或页面上

08. 总并发请求数指什么？
同一时间在系统上的用户同时向服务器做出的请求数量，这些请求也可能分布在不同的功能模块或
页面上。

09. 平均吞吐量指什么？
吞吐量表示待测应用对业务的支持量，以 TPS 或 QPS 为单位，表示每秒能处理的请求数

10. 平均响应时间指什么？
一些请求从发起到收到服务端响应所需要的时间的平均数

11. 如何计算平均响应时间和平均吞吐量
一段时间内的请求平均响应时间 = 单个请求的响应时间之和（秒）/这段时间内服务端处理掉的总请求数
一段时间内的平均吞吐量 = 这段时间内服务端处理掉的总请求数 / 时间（秒）

12. TPS/QPS 代表什么意思
TPS：transaction per second
QPS：Query per second

13. 推导平均吞吐量和平均响应时间的关系
这段时间服务端处理掉的总请求数为n(个)
这段时间为t(秒)
一段时间内请求平均响应为 Tv
一段时间内平均吞吐量为 Qv
单个响应的响应时间为 Ts
SUM为求和函数
Tv = SUM(Ts) / n
Qv = n / t
推导出 Qv = SUM(Ts)/Tv/t
Qv = SUM(Ts)/Tv·t
说明平均吞吐量 Qv 与平均响应时间 Tv 成反比


14. 是否可以通过吞吐量来判断不同业务的服务端性能?
    不能，吞吐量与具体业务密切相关，业务耗时小的服务器的吞吐量显然比业务耗时大的
    服务器吞吐量高。

15. 错误率的定义
错误率 = 一段时间内出错请求个数 / 总请求数

16. 什么叫出错的请求？
1）没有任何响应的请求 - 超时
2）有响应，但是卡了很久才给响应 - 超时
3）有响应，响应代码不是预期值 - 断言错误
4）有相应，响应代码是预期值，但响应体内有数据不是预期值 - 断言错误

17. 超时请求有什么影响？
根据平均响应时间的公式，单个请求的响应时间之和中包含了超时请求，所以超时请求会导致
平均响应时间变长

18. 90% 平均响应时间是什么意思？
从平均响应时间计算所统计的那些请求里，去掉响应时间最长的10%的请求后，剩余的请求
计算出来的平均响应时间

19. 数据吞吐量/平均传输带宽是用来干嘛的？
用于计算服务端的数据传输质量
